# Claude Desktop Windows 简体中文汉化工具

这是一个给 Windows 版 Claude Desktop 使用的简体中文本地汉化工具。

它的目标很简单：尽量让 Claude Desktop 的菜单、设置、提示、连接器、第三方推理配置、遥测说明、错误提示等界面内容显示为自然一点的中文，同时尽量保留 Claude Desktop 原本的使用方式，尤其是 Cowork 功能的兼容性。

本项目不是 Claude 官方项目，也不包含 Claude 官方程序文件。使用前需要你已经在电脑上安装过官方 Windows 版 Claude Desktop。

## 主要特点

- 一键安装简体中文界面。
- 安装后会创建 `Claude zh-CN` 快捷方式，之后从这个快捷方式启动中文版。
- 简体中文会出现在 Language 语言列表的第一个选项。
- 覆盖常见设置页、连接器和扩展、第三方推理配置、遥测和更新、外观、错误提示等文本。
- 使用固定安装位置，避免桌面生成一堆重复的 Claude 文件夹。
- 保留 `WindowsApps` 路径结构，尽量兼容 Cowork。
- 支持自己手动调整翻译，不满意的中文可以改。
- 重复运行会复用缓存和备份，通常会比第一次快很多。
- 附带 Cowork 工作区权限修复脚本，用来处理部分电脑上的 `HCS 0x80070005` / `rootfs.vhdx 拒绝访问` 问题。

## 支持环境

- Windows 10 / Windows 11。
- 已安装官方 Windows 版 Claude Desktop。
- 系统自带 PowerShell 即可。

暂不支持 macOS、Linux，也不适用于网页版 Claude。

## 下载和安装

1. 在 Releases 里下载压缩包。
2. 解压到任意文件夹，建议不要放在系统盘根目录或需要特殊权限的位置。
3. 关闭正在运行的 Claude Desktop。
4. 双击 `一键安装汉化.cmd`。
5. 如果 Windows 弹出权限确认，选择允许。
6. 等待脚本完成。
7. 从桌面或开始菜单里的 `Claude zh-CN` 快捷方式启动。

安装完成后，汉化版会放在当前用户目录下：

`%LOCALAPPDATA%\Programs\Claude-zh-CN`

请不要移动或重命名这个目录里的 `WindowsApps`、`Claude_版本_x64__pzs8sxrjxfjjc`、`app` 等文件夹。Cowork 会检查运行路径，随意移动可能会导致它提示需要重新安装桌面应用。

## 文件说明

`一键安装汉化.cmd`

安装或重新安装简体中文汉化。

`恢复英文.cmd`

把已安装的汉化版恢复为英文界面。

`修复Cowork工作区权限.cmd`

当 Cowork 报 `HCS 0x80070005`、`rootfs.vhdx 拒绝访问`、无法启动工作区时，可以先关闭 Claude，再运行这个脚本修复权限。

`手动修改翻译.json`

给用户自己改翻译用。界面上某个中文觉得不自然，可以在这里搜索并修改。

`翻译对照表.json`

翻译参考表，里面可以看到英文原文和当前中文。这个文件主要用来查，不建议直接当作修改入口。

`使用教程.txt`

更详细的普通用户教程。

`程序文件-不要改名`

汉化脚本和语言文件所在目录。不要改名，也不要删除里面的文件。

## 自己修改翻译

如果你看到界面里某句中文不合适，可以这样改：

1. 打开 `手动修改翻译.json`。
2. 搜索你在 Claude 界面里看到的中文。
3. 只修改冒号右边双引号里的中文内容。
4. 保存文件。
5. 重新运行 `一键安装汉化.cmd`。
6. 重新打开 `Claude zh-CN`。

如果你不知道这句中文对应的英文原文是什么，可以打开 `翻译对照表.json` 搜索同一句中文或附近关键词作为参考。

不建议直接修改 `程序文件-不要改名` 里的核心脚本，除非你知道自己在改什么。

## 关于原版 Claude

建议保留官方原版 Claude Desktop。

这个工具会复制并生成一个独立的汉化运行目录，不是把官方安装目录直接改坏。即使汉化版可以单独启动，原版 Claude 仍然建议保留，因为官方更新、重新复制新版、部分系统入口和 Cowork 修复都可能需要它。

如果卸载了原版，已经生成的 `Claude zh-CN` 通常仍然可以继续运行，但后续更新和修复会更麻烦。所以更稳妥的做法是：原版保留，日常使用中文版快捷方式。

## 常见问题

### 打开后还是英文

请确认你启动的是 `Claude zh-CN` 快捷方式，而不是原版 Claude 入口。

如果仍然是英文，关闭 Claude 后重新运行 `一键安装汉化.cmd`。

### 关闭到托盘后，双击快捷方式没有弹出窗口

这是 Claude Desktop 自身的托盘行为。可以先从托盘图标打开窗口，或者右键退出 Claude 后再重新打开。

### Cowork 提示需要重新安装桌面应用

不要移动汉化版安装目录。先关闭 Claude，然后重新运行 `一键安装汉化.cmd`。

这个版本会把汉化版固定安装到 `%LOCALAPPDATA%\Programs\Claude-zh-CN`，并保留 `WindowsApps` 路径结构，就是为了解决 Cowork 对路径比较敏感的问题。

### Cowork 报 HCS 0x80070005 或 rootfs.vhdx 拒绝访问

这通常是 Windows 虚拟机权限问题，原版 Claude 也可能出现。

处理方法：

1. 关闭 Claude。
2. 运行 `修复Cowork工作区权限.cmd`。
3. 重新打开 `Claude zh-CN`。

如果仍然不行，可以重启电脑后再试。

### 每次运行都会备份吗

工具会在当前汉化工具文件夹里保留 `Claude备份`，用于出问题时恢复参考。新版会尽量复用已有备份，不会像早期版本那样一直在桌面生成很多带时间的备份文件夹。

### 可以删除这个汉化工具文件夹吗

安装完成后，中文版实际运行目录在 `%LOCALAPPDATA%\Programs\Claude-zh-CN`。如果你已经确认中文界面正常，删除下载解压出来的汉化工具文件夹后，`Claude zh-CN` 通常仍然可以继续运行。

但如果你以后还想修改翻译、恢复英文、重新安装或修复 Cowork 权限，建议保留这个工具文件夹。

## 发布到 GitHub 时请注意

本项目只应该发布汉化工具本身，不要上传 Claude 官方程序。

请不要上传这些内容：

- `Claude备份`
- `Claude-zh-CN`
- `Claude-zh-CN-时间`
- 官方 Claude 的 `app` 目录
- `claude.exe`
- `app.asar`
- `.dll`
- `.pak`
- `.vhdx`
- 任何从官方 Claude 安装目录直接复制出来的大文件

仓库里放工具文件和翻译文件即可。Release 压缩包也一样，不要把本机生成的 Claude 程序副本打进去。

如果公开发布时使用了其他开源项目中的翻译内容，请先确认对方许可证允许复用，或取得作者授权。
